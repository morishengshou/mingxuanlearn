在 Java **Stream API** 里，常把操作分成两类：**中间操作（Intermediate Operations）** 和 **终端操作（Terminal Operations）**。你说的“中间操作方法类/终端操作方法类”，一般就是指这两类方法。

---

## 1) 中间操作（中间操作方法）
**定义**：对 Stream 做“加工/变换”，并**返回一个新的 Stream**，用于继续拼接流水线。

**核心特征**
- **返回类型通常是 `Stream` / `IntStream` / `LongStream` / `DoubleStream`**（仍然是流）。
- **惰性执行（lazy）**：调用了中间操作并不会立刻遍历数据；只有遇到终端操作才会真正执行。
- 主要用于：过滤、映射、排序、去重、截断、展开等。

**常见中间操作**
- `filter`
- `map` / `mapToInt`…
- `flatMap`
- `distinct`
- `sorted`
- `peek`（调试用，谨慎用于副作用）
- `limit` / `skip`
- `takeWhile` / `dropWhile`（Java 9+）

---

## 2) 终端操作（终端操作方法）
**定义**：触发对 Stream 的实际遍历计算，并**产生最终结果或副作用**；执行后流水线结束。

**核心特征**
- **返回类型不是 Stream**，而是：
  - 一个结果：如 `long`、`Optional<T>`、集合、字符串等
  - 或者 `void`（如 `forEach`）
- **立即执行（eager，触发执行）**：一旦调用，整个流水线开始跑。
- **一个 Stream 只能执行一次终端操作**；执行完就“消费掉”了，再用会抛 `IllegalStateException`。

**常见终端操作**
- 遍历/副作用：`forEach` / `forEachOrdered`
- 归约：`reduce`
- 收集：`collect`（如 `Collectors.toList()` / `toMap()`）
- 统计：`count`、`min`、`max`
- 匹配：`anyMatch` / `allMatch` / `noneMatch`
- 查找：`findFirst` / `findAny`
- 数组：`toArray`

---

## 3) 主要区别总结
| 维度 | 中间操作 | 终端操作 |
|---|---|---|
| 返回值 | 仍是 Stream（可继续链式调用） | 非 Stream（得到结果或 void） |
| 是否触发执行 | 不触发（惰性） | 触发（开始遍历计算） |
| 对 Stream 的影响 | 构建流水线 | 消费流水线、结束 Stream |
| 调用次数 | 可多个串联 | 通常只能有一个（每条流水线） |

---

## 4) 例子：看出“惰性 + 触发”
```java
List<String> list = List.of("a", "bb", "ccc");

Stream<String> s = list.stream()
    .filter(x -> {
        System.out.println("filter: " + x);
        return x.length() >= 2;
    })
    .map(x -> {
        System.out.println("map: " + x);
        return x.toUpperCase();
    }); // 到这里都不会打印（中间操作惰性）

long cnt = s.count(); // 终端操作触发执行，这里才开始打印并计算
System.out.println(cnt);
```

---

## 5) 补充：短路（常考点）
某些终端操作会**短路**，不一定遍历完所有元素，比如：
- `anyMatch`、`findFirst`、`findAny`
以及配合某些中间操作如 `limit` 也会让处理提前停止。
