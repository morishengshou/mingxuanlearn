1) JVM 运行时内存区域（更常被问“分为几个区”）
根据《Java 虚拟机规范》，运行时数据区主要分 5 个：

程序计数器（Program Counter Register）

每个线程私有，记录当前线程执行到的字节码行号/位置。
Java 虚拟机栈（JVM Stack）

每个线程私有，栈帧包含：局部变量表、操作数栈、动态链接、方法返回地址等。
本地方法栈（Native Method Stack）

每个线程私有，为执行 JNI 等本地方法服务（有的 JVM 与虚拟机栈合并实现）。
堆（Heap）

线程共享，对象实例主要分配在这里，是 GC 管理的核心区域。
方法区（Method Area）

线程共享，存放类元数据、运行时常量池等。
在 HotSpot 中，JDK 8 起很多“类元信息”迁移到 Metaspace（元空间）（使用本地内存实现），但规范层面仍称“方法区”这个概念。
补充：运行时常量池通常被视为方法区的一部分（规范里是方法区的组成），有些资料会单独列出来，但严格说仍归入方法区范畴。
