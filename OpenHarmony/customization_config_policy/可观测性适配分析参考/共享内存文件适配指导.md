# config_policy 共享内存文件替代系统参数适配指导

## 适用场景

移植目标系统**既无系统参数/属性系统**（如 OpenHarmony `param get` / Android `getprop`），
又**无法修改 init 进程代码**，导致环境变量方案生效范围受限时，
使用 tmpfs 文件或 POSIX 共享内存文件作为参数存储载体。

---

## 一、为什么环境变量生效范围有限

环境变量是**进程私有**的，子进程通过 `fork()` 从父进程继承。
若无法修改 init 进程，就无法在系统启动时向所有进程注入环境变量——
只有 init `fork` 出的进程才能继承，init 自身的环境你无法控制。

**tmpfs 文件**解决了这一问题：它是文件系统上的一个路径，
任何拥有读权限的进程都可以打开，与进程亲缘关系无关。

### 关于 procfs 的说明

`/proc` 是内核管理的只读文件系统，仅展示内核数据结构，
**不支持用户自定义 key-value 存储**（除非编写内核模块）。
用户态参数存储应选 tmpfs，而非 procfs。

---

## 二、与其他适配方案的关系

本文档是 `移植指导.md` 中"适配方案 A（同名头文件包装）"的第二种具体实现，
与 `环境变量适配指导.md` 并列，选其中一种即可。

| 方案 | 适配文件 | 适用条件 |
|------|----------|----------|
| 环境变量 | `环境变量适配指导.md` | 可控制进程启动环境 |
| **tmpfs 文件（本文档）** | **本文档** | **不能改 init，需系统级共享** |

两种方案的 `config_policy_utils.c` 改动完全相同（仅 2 处 `#include`），
只需替换 `port/config_policy_param_adapter.h` 的内容。

## 三、方案A：tmpfs 文本文件（推荐）

### 3.1 参数文件格式

文件路径：`/run/config_policy/params`（可编译期自定义）

```ini
# config_policy 参数文件，由系统启动脚本写入
# 格式：key=value，每行一条，'#' 开头为注释
const.cust.config_dir_layer=/system:/vendor:/oem
const.cust.follow_x_rules=
telephony.sim.opkey0=46060
telephony.sim.opkey1=46061
```

规则：
- `key=value` 格式，等号两侧无空格
- `#` 开头的行为注释，忽略
- 空行忽略
- 同一 key 出现多次时取第一次出现的值

### 3.2 写入方（启动脚本）

无需修改 init 本身，在**比 init 晚启动的某个服务脚本**中写入即可。
使用原子替换（先写临时文件再 `mv`）防止读写竞争：

```bash
#!/bin/sh
# /etc/init.d/config-policy-params 或 systemd ExecStartPre 脚本

mkdir -p /run/config_policy

# 原子写入：先写临时文件，写完再替换，防止读到半写文件
cat > /run/config_policy/params.tmp << 'EOF'
# config_policy 参数
const.cust.config_dir_layer=/system:/vendor:/oem
const.cust.follow_x_rules=
telephony.sim.opkey0=46060
telephony.sim.opkey1=46061
EOF

mv /run/config_policy/params.tmp /run/config_policy/params

# 立即设为只读，写入与权限设置之间无间隙
chmod 444 /run/config_policy/params
chown root:root /run/config_policy/params
```

### 3.3 完整适配头文件

新建 `port/config_policy_param_adapter.h`：

```c
/*
 * port/config_policy_param_adapter.h
 *
 * 用 tmpfs 文本文件替代 OpenHarmony init_param.h，
 * 适用于无系统参数/属性系统且无法修改 init 进程的移植目标平台。
 *
 * 参数文件路径可在编译期通过 -DCONFIG_POLICY_PARAM_FILE="..." 覆盖。
 */
#ifndef CONFIG_POLICY_PARAM_ADAPTER_H
#define CONFIG_POLICY_PARAM_ADAPTER_H

#include <stdio.h>
#include <string.h>
#include <stdlib.h>

#ifdef __cplusplus
extern "C" {
#endif

/* 参数文件路径，可通过 -D 编译选项覆盖 */
#ifndef CONFIG_POLICY_PARAM_FILE
#  define CONFIG_POLICY_PARAM_FILE "/run/config_policy/params"
#endif

/*
 * PARAM_CONST_VALUE_LEN_MAX
 * 与 OH 原始定义兼容，源码用它限制动态缓冲区上限。
 * OH 原始值为 96，此处扩大到 512 以适应更长的自定义分层路径。
 */
#ifndef PARAM_CONST_VALUE_LEN_MAX
#  define PARAM_CONST_VALUE_LEN_MAX 512
#endif

/* 行缓冲：key 最长 128 + '=' + value 最长 512 + '\n\0' */
#define _PARAM_LINE_BUF (128 + 1 + PARAM_CONST_VALUE_LEN_MAX + 2)

/* ------------------------------------------------------------------ */
/* SystemGetParameter                                                   */
/* ------------------------------------------------------------------ */

/**
 * 从参数文件中查找 key 对应的 value。
 * 接口语义与 OpenHarmony init_param.h 完全一致：
 *   value == NULL : 将所需缓冲区字节数（含 '\0'）写入 *len，返回 0
 *   value != NULL : 将值复制进 value，返回 0
 *   未找到 / 文件不存在 : 返回 -1（模块将回落到编译期 DEFAULT_LAYER）
 */
static inline int SystemGetParameter(const char *name,
                                      char *value,
                                      unsigned int *len)
{
    if (name == NULL || len == NULL) return -1;

    FILE *f = fopen(CONFIG_POLICY_PARAM_FILE, "r");
    if (f == NULL) return -1; /* 文件不存在，触发 DEFAULT_LAYER 回落 */

    char line[_PARAM_LINE_BUF];
    size_t name_len = strlen(name);
    char found_val[PARAM_CONST_VALUE_LEN_MAX];
    int found = 0;

    while (fgets(line, (int)sizeof(line), f) != NULL) {
        /* 跳过注释行和空行 */
        if (line[0] == '#' || line[0] == '\n' || line[0] == '\r') continue;

        /* 检查前缀匹配："name=" */
        if (strncmp(line, name, name_len) != 0) continue;
        if (line[name_len] != '=') continue;

        /* 提取 value，裁剪尾部换行符 */
        char *val = line + name_len + 1;
        size_t vlen = strlen(val);
        while (vlen > 0 &&
               (val[vlen - 1] == '\n' || val[vlen - 1] == '\r')) {
            vlen--;
        }
        /* 防截断：不应超过 PARAM_CONST_VALUE_LEN_MAX - 1 */
        if (vlen >= PARAM_CONST_VALUE_LEN_MAX) {
            vlen = PARAM_CONST_VALUE_LEN_MAX - 1;
        }
        memcpy(found_val, val, vlen);
        found_val[vlen] = '\0';
        found = 1;
        break; /* 同名 key 取第一条 */
    }
    fclose(f);

    if (!found) return -1;

    unsigned int total = (unsigned int)(strlen(found_val) + 1);
    if (value == NULL) {
        *len = total; /* 第 1 步调用：仅返回所需长度 */
        return 0;
    }
    if (*len < total) return -1; /* 调用方缓冲区不足 */
    memcpy(value, found_val, total);
    return 0;
}

/* ------------------------------------------------------------------ */
/* SystemSetParameter（可选，仅用于测试时动态注入参数）                */
/* ------------------------------------------------------------------ */

/**
 * 运行期向参数文件追加或覆盖一条记录。
 * 注意：文件设为只读（chmod 444）后此函数会失败，这是预期行为。
 * 主要用于单元测试场景，生产环境通过启动脚本写入。
 */
static inline int SystemSetParameter(const char *name, const char *value)
{
    if (name == NULL || value == NULL) return -1;
    FILE *f = fopen(CONFIG_POLICY_PARAM_FILE, "a"); /* 追加模式 */
    if (f == NULL) return -1;
    fprintf(f, "%s=%s\n", name, value);
    fclose(f);
    return 0;
}

#ifdef __cplusplus
}
#endif

#undef _PARAM_LINE_BUF
#endif /* CONFIG_POLICY_PARAM_ADAPTER_H */
```

## 四、方案B：POSIX 共享内存 + mmap

适用于**高频读取场景**（毫秒级循环调用配置查询），
通过内存映射避免重复 `fopen`，提升读取性能。

对于 config_policy 的典型使用（程序启动时查询数次），方案A 已足够，
方案B 适合有性能要求的场景。

### 4.1 写入方

```c
/*
 * 写入 POSIX 共享内存，在系统启动时由有写权限的进程调用一次。
 * content 格式与方案A 的参数文件完全相同（key=value 文本）。
 */
#include <fcntl.h>
#include <sys/mman.h>
#include <sys/stat.h>
#include <unistd.h>
#include <string.h>
#include <stdio.h>

#define SHM_NAME "/config_policy_params"

int WriteConfigPolicyShm(const char *content)
{
    size_t size = strlen(content) + 1;

    /* 以 0444 权限创建，写入后自然只读 */
    int fd = shm_open(SHM_NAME, O_CREAT | O_EXCL | O_RDWR, 0444);
    if (fd < 0) return -1;

    if (ftruncate(fd, (off_t)size) < 0) {
        close(fd);
        shm_unlink(SHM_NAME);
        return -1;
    }

    void *ptr = mmap(NULL, size, PROT_READ | PROT_WRITE, MAP_SHARED, fd, 0);
    if (ptr == MAP_FAILED) {
        close(fd);
        shm_unlink(SHM_NAME);
        return -1;
    }

    memcpy(ptr, content, size);
    munmap(ptr, size);
    close(fd);
    return 0;
}
```

### 4.2 读取方适配头文件

```c
/* port/config_policy_param_adapter.h（方案B 版本） */
#ifndef CONFIG_POLICY_PARAM_ADAPTER_H
#define CONFIG_POLICY_PARAM_ADAPTER_H

#include <fcntl.h>
#include <sys/mman.h>
#include <sys/stat.h>
#include <unistd.h>
#include <string.h>

#ifdef __cplusplus
extern "C" {
#endif

#ifndef CONFIG_POLICY_SHM_NAME
#  define CONFIG_POLICY_SHM_NAME "/config_policy_params"
#endif

#ifndef PARAM_CONST_VALUE_LEN_MAX
#  define PARAM_CONST_VALUE_LEN_MAX 512
#endif

static inline int SystemGetParameter(const char *name,
                                      char *value,
                                      unsigned int *len)
{
    if (name == NULL || len == NULL) return -1;

    int fd = shm_open(CONFIG_POLICY_SHM_NAME, O_RDONLY, 0);
    if (fd < 0) return -1;

    struct stat st;
    if (fstat(fd, &st) < 0) { close(fd); return -1; }

    size_t map_size = (size_t)st.st_size;
    const char *base = (const char *)mmap(NULL, map_size,
                                           PROT_READ, MAP_SHARED, fd, 0);
    close(fd);
    if (base == MAP_FAILED) return -1;

    /* 在内存映射中逐行扫描（与方案A 解析逻辑相同） */
    size_t name_len = strlen(name);
    char found_val[PARAM_CONST_VALUE_LEN_MAX];
    int found = 0;
    const char *p = base;
    const char *end = base + map_size;

    while (p < end) {
        /* 找到本行结尾 */
        const char *eol = (const char *)memchr(p, '\n', (size_t)(end - p));
        size_t line_len = eol ? (size_t)(eol - p) : (size_t)(end - p);

        if (line_len > 0 && p[0] != '#') {
            if (line_len > name_len && strncmp(p, name, name_len) == 0
                && p[name_len] == '=') {
                const char *val = p + name_len + 1;
                size_t vlen = line_len - name_len - 1;
                /* 裁剪 '\r' */
                if (vlen > 0 && val[vlen - 1] == '\r') vlen--;
                if (vlen >= PARAM_CONST_VALUE_LEN_MAX)
                    vlen = PARAM_CONST_VALUE_LEN_MAX - 1;
                memcpy(found_val, val, vlen);
                found_val[vlen] = '\0';
                found = 1;
                break;
            }
        }
        p = eol ? eol + 1 : end;
    }

    munmap((void *)base, map_size);

    if (!found) return -1;

    unsigned int total = (unsigned int)(strlen(found_val) + 1);
    if (value == NULL) { *len = total; return 0; }
    if (*len < total) return -1;
    memcpy(value, found_val, total);
    return 0;
}

static inline int SystemSetParameter(const char *name, const char *value)
{
    (void)name; (void)value;
    return -1; /* shm 对象以只读模式创建，不支持运行期写入 */
}

#ifdef __cplusplus
}
#endif
#endif /* CONFIG_POLICY_PARAM_ADAPTER_H */
```

## 五、两种方案对比

| 对比项 | 方案A：tmpfs 文本文件 | 方案B：POSIX shm + mmap |
|--------|----------------------|------------------------|
| 实现复杂度 | 低 | 中 |
| 每次查询开销 | `fopen` + 顺序扫描 | `shm_open` + `mmap` + 扫描 |
| 适合调用频率 | 启动期少量调用 | 高频调用 |
| 文件路径 | `/run/config_policy/params` | `/dev/shm/config_policy_params` |
| 额外链接库 | 无 | `-lrt` |
| 调试友好性 | `cat /run/config_policy/params` 直接查看 | 需要读取程序才能查看内容 |
| 推荐程度 | **推荐（大多数场景）** | 按需选用 |

## 六、只读权限配置

### 6.1 文件权限位（最常用）

```bash
chmod 444 /run/config_policy/params   # 所有用户只读，无人可写
# 或仅指定组可读
chmod 640 /run/config_policy/params
chown root:config_policy /run/config_policy/params
```

进程只需持有对该文件的 `r` 权限即可读取，完全无法修改内容。

### 6.2 原子写入 + 立即锁定

写入与 `chmod` 之间不应有时间窗口，使用 `mv` 原子替换：

```bash
# 写入临时文件
cat > /run/config_policy/params.tmp << 'EOF'
const.cust.config_dir_layer=/system:/vendor:/oem
EOF

# 原子替换（mv 在同一文件系统内是原子操作）
mv /run/config_policy/params.tmp /run/config_policy/params

# 立即设为只读
chmod 444 /run/config_policy/params
```

### 6.3 Linux 文件封印（防止 root 修改）

使用 `memfd_create` + `F_ADD_SEALS`，适合安全要求极高的场景：

```c
#include <sys/memfd.h>
#include <linux/fcntl.h>

/* 创建匿名内存文件 */
int fd = memfd_create("config_policy_params", MFD_ALLOW_SEALING);
ftruncate(fd, size);

void *ptr = mmap(NULL, size, PROT_READ | PROT_WRITE, MAP_SHARED, fd, 0);
memcpy(ptr, content, size);
munmap(ptr, size);

/* 封印：之后任何进程（包括 root）都无法再写入或调整大小 */
fcntl(fd, F_ADD_SEALS,
      F_SEAL_SHRINK | F_SEAL_GROW | F_SEAL_WRITE | F_SEAL_SEAL);

/*
 * 通过 /proc/self/fd/<fd> 路径让其他进程以只读方式打开，
 * 或通过 Unix domain socket 传递 fd（需要接收方用 SCM_RIGHTS）。
 */
```

> `memfd` 封印后，即使是 `root` 也无法绕过（除非持有 `CAP_SYS_ADMIN` 且内核允许）。
> 适合安全敏感场景，普通嵌入式设备 `chmod 444` 已足够。

### 6.4 目录级只读（tmpfs remount）

若要保护整个参数目录，写入完成后重新以只读挂载：

```bash
mount --bind /run/config_policy /run/config_policy
mount -o remount,ro,bind /run/config_policy
```

之后该目录下所有文件对所有进程只读，直到系统重启或手动 `remount rw`。

## 八、风险点与缓解措施

### 8.1 启动竞争条件（最重要）

**风险**：参数文件由某个启动服务写入，但 config_policy 可能在文件写完之前就被调用。
`fopen` 返回 `NULL`，适配层回落到 `DEFAULT_LAYER`，进程使用错误的层级配置。

**缓解措施**：

在 systemd 中声明依赖顺序：
```ini
# /etc/systemd/system/your-service.service
[Unit]
After=config-policy-params.service
Requires=config-policy-params.service
```

或在启动脚本中等待文件出现：
```bash
while [ ! -f /run/config_policy/params ]; do
    sleep 0.1
done
```

或在适配层加入简单重试（不超过 1 秒）：
```c
for (int i = 0; i < 10 && f == NULL; i++) {
    usleep(100000); /* 等待 100ms */
    f = fopen(CONFIG_POLICY_PARAM_FILE, "r");
}
```

### 8.2 tmpfs 未挂载

**风险**：部分极简系统（无 systemd、busybox initrd）`/run` 可能不存在或不是 tmpfs。

**缓解措施**：将路径改为 `/dev/shm/config_policy_params`，
`/dev/shm` 在几乎所有 Linux 系统上都是 tmpfs；
或通过 `-DCONFIG_POLICY_PARAM_FILE=...` 指向已知可写的目录。

### 8.3 重启后数据丢失

**风险**：tmpfs 是 RAM 文件系统，断电或重启后参数文件消失，
**必须在每次开机时重新写入**。若启动脚本未执行（如安全模式），
参数文件不存在，模块静默使用 `DEFAULT_LAYER`。

**缓解措施**：将参数写入脚本加入 init 的必选启动序列；
同时在系统日志中记录"使用默认层级"警告，便于排查。

### 8.4 文件解析错误

**风险**：文件在写入过程中被读取（未使用原子替换），或文件内容损坏，
导致 `SystemGetParameter` 找不到 key，静默回落 `DEFAULT_LAYER`，难以排查。

**缓解措施**：**始终使用原子写入**（先写 `.tmp`，再 `mv` 替换）；
启动脚本完成后用 `dump_config_layers` 工具验证参数文件内容正确。

### 8.5 权限配置不当

**风险**：若文件为 `world-writable`（0666），任何进程都能篡改配置层路径，造成配置劫持。

**缓解措施**：**写入与 `chmod 444` 之间不允许有时间窗口**，
使用原子替换（`mv`）配合目标文件直接以 444 创建：
```bash
install -m 444 /tmp/params.tmp /run/config_policy/params
```

### 8.6 每次查询重新解析（性能）

**风险**：方案A 每次 `SystemGetParameter` 调用都执行 `fopen` + 顺序扫描，
在启动期少量调用时可接受，在高频场景下会产生性能开销。

**缓解措施**：增加进程内一次性缓存（读取后存入静态变量）：

```c
/* 仅在第一次调用时读取文件，后续返回缓存值 */
static char g_layer_cache[PARAM_CONST_VALUE_LEN_MAX] = {0};
static int  g_layer_valid = 0; /* 0=未缓存，1=已缓存 */
```

> 注意：缓存后若参数文件被外部更新，进程内不会感知变化。
> 对于只读不变的参数文件，这是可接受的权衡。

### 8.7 文件路径长度

**风险**：`CONFIG_POLICY_PARAM_FILE` 若路径本身很长，可能超出 `PATH_MAX`。

**缓解措施**：保持路径在 64 字符以内（`/run/config_policy/params` = 27 字符）；
若需要更长路径，在 `fopen` 前检查 `strlen(path) < PATH_MAX`。
