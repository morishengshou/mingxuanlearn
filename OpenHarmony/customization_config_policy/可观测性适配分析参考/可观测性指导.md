# config_policy 配置分层路径可观测性指导

## 概述

本文档说明 `customization_config_policy` 模块在配置分层路径方面的可观测性现状，并提供在移植场景下补齐可观测能力的完整指导。

---

## 一、现有可观测性能力

### 1.1 能力总览

| 观测手段 | 是否存在 | 能否看到实际生效路径 | 适用场景 |
|----------|----------|----------------------|----------|
| `param get const.cust.config_dir_layer` | **存在** | 部分（见盲区说明） | 设备 shell 调试 |
| 单元测试二进制 `ConfigPolicyUtilsTest` | **存在** | 是（打印解析后路径） | 开发调试环境 |
| `getCfgDirListSync()` JS/NAPI 接口 | **存在** | 是 | 应用层代码调用 |
| `CJ_GetCfgDirList()` Cangjie FFI 接口 | **存在** | 是 | Cangjie 语言调用 |
| `hidumper` 支持 | **不存在** | — | — |
| SA 服务 Dump 接口 | **不存在** | — | — |
| 专用 CLI 工具 | **不存在** | — | — |
| HiSysEvent 路径内容上报 | **不存在** | 否（仅上报 API 名称和错误码） | — |

### 1.2 方式一：读取系统参数

```bash
hdc shell param get const.cust.config_dir_layer
```

**输出示例：**

```
/system:/chipset:/sys_prod:/chip_prod
```

**说明：**

- 该参数即 `GetCfgDirRealPolicyValue()` 所读取的原始值，冒号分隔各层路径。
- 参数 key 定义于 `interfaces/inner_api/include/config_policy_impl.h`：
  ```c
  #define CUST_KEY_POLICY_LAYER "const.cust.config_dir_layer"
  ```
- **盲区**：若该参数在参数系统中未设置，`param get` 返回空，但模块内部会静默回落到编译期硬编码的默认值（见下文 1.4 节），实际生效路径对外完全不可见。

同理，Follow-X 规则参数也可查询：

```bash
hdc shell param get const.cust.follow_x_rules
```

### 1.3 方式二：运行单元测试（调试环境）

```bash
hdc shell /data/test/ConfigPolicyUtilsTest --gtest_filter=CfgPolicyUtilsFuncTest005
```

`CfgPolicyUtilsFuncTest005`（位于 `test/unittest/config_policy_utils_test.cpp:122`）内部调用 `GetCfgDirList()` 并逐条打印路径，是目前唯一能看到**解析后实际生效路径**的手段。

**输出示例：**

```
filePath: /system
filePath: /chipset
filePath: /sys_prod
filePath: /chip_prod
```

### 1.4 参数未设置时的静默 fallback（盲区）

`frameworks/config_policy/src/config_policy_utils.c:405`：

```c
// 若系统参数不存在，直接使用编译期默认值，不产生任何日志或错误
res->realPolicyValue = strdup(DEFAULT_LAYER);
```

`DEFAULT_LAYER` 定义于 `interfaces/inner_api/include/config_policy_impl.h:37`：

```c
#define DEFAULT_LAYER \
    ROOT_PREFIX "/system:" ROOT_PREFIX "/chipset:" \
    ROOT_PREFIX "/sys_prod:" ROOT_PREFIX "/chip_prod"
```

**影响：** 在参数系统层面无法区分"参数已配置且值为默认路径"与"参数未配置，代码使用编译期默认"这两种状态。

## 二、现有能力的不足

1. **无专用 CLI 工具**：整个仓库没有 `ohos_executable` 构建目标，无法直接在 shell 中执行一条命令获取解析后的实际生效路径。

2. **无 hidumper 支持**：未实现 `IDump` 接口，无法通过 `hidumper -s` 查询。

3. **无 SA 服务**：模块是纯库，无服务注册（无 `sa_profile`），不具备服务级别的 dump 能力。

4. **HiSysEvent 不含路径信息**：`frameworks/dfx/hisysevent.yaml` 中定义的两个事件（`CONFIG_POLICY_FAILED`、`CONFIG_POLICY_EVENT`）只上报 API 名称，不含路径值。

5. **fallback 不透明**：参数未设置时，模块静默使用编译期默认值，无任何日志或状态标记，难以判断当前配置来源。

---

## 三、移植时补齐可观测性

移植到其他 Harmony 内核系统后，建议主动补齐以下能力。`GetCfgDirList()` 已将所有信息整理好，只需提供调用入口。

### 3.1 方案一：简单 CLI 查询工具（推荐，必做）

**新建文件 `tools/dump_config_layers.c`：**

```c
#include <stdio.h>
#include "config_policy_utils.h"

int main(void)
{
    CfgDir *dirs = GetCfgDirList();
    if (dirs == NULL) {
        fprintf(stderr, "GetCfgDirList() returned NULL\n");
        return 1;
    }

    printf("=== Config Layer Paths ===\n");
    /* realPolicyValue 直接暴露实际生效值，
     * 无论来自参数系统还是编译期默认值，此处均可见。*/
    printf("Raw policy value : %s\n",
           dirs->realPolicyValue ? dirs->realPolicyValue : "(empty)");
    printf("Parsed paths (priority: low -> high):\n");
    for (size_t i = 0; i < MAX_CFG_POLICY_DIRS_CNT; i++) {
        if (dirs->paths[i] != NULL && dirs->paths[i][0] != '\0') {
            printf("  [%zu] %s\n", i, dirs->paths[i]);
        }
    }

    FreeCfgDirList(dirs);
    return 0;
}
```

**CMakeLists.txt 新增：**

```cmake
add_executable(dump_config_layers tools/dump_config_layers.c)
target_link_libraries(dump_config_layers config_policy_util)
target_include_directories(dump_config_layers PRIVATE
    interfaces/inner_api/include
)
```

**运行效果：**

```
=== Config Layer Paths ===
Raw policy value : /system:/chipset:/sys_prod:/chip_prod
Parsed paths (priority: low -> high):
  [0] /system
  [1] /chipset
  [2] /sys_prod
  [3] /chip_prod
```

> **为何不直接用 `param get`？**
> 直接读取 `realPolicyValue` 字段绕过了 1.4 节描述的盲区——无论路径来自参数系统还是编译期默认值，都能被观测到，是目前最可靠的观测方式。

### 3.2 方案二：完整诊断工具（按需选做）

在方案一基础上扩展，支持同时诊断指定文件的查找结果，适合排查"为什么用的不是预期的配置文件"这类问题。

**`tools/dump_config_layers.c`（完整版）：**

```c
#include <stdio.h>
#include <string.h>
#include "config_policy_utils.h"

static void dump_layers(void)
{
    CfgDir *dirs = GetCfgDirList();
    if (dirs == NULL) {
        fprintf(stderr, "GetCfgDirList() returned NULL\n");
        return;
    }
    printf("=== Config Layer Paths ===\n");
    printf("Raw policy value : %s\n",
           dirs->realPolicyValue ? dirs->realPolicyValue : "(empty — using compile-time default)");
    printf("Parsed paths (priority: low -> high):\n");
    for (size_t i = 0; i < MAX_CFG_POLICY_DIRS_CNT; i++) {
        if (dirs->paths[i] != NULL && dirs->paths[i][0] != '\0') {
            printf("  [%zu] %s\n", i, dirs->paths[i]);
        }
    }
    FreeCfgDirList(dirs);
}

static void dump_file_lookup(const char *rel_path)
{
    static const struct { int mode; const char *name; } modes[] = {
        { FOLLOWX_MODE_NO_RULE_FOLLOWED, "NO_RULE_FOLLOWED" },
        { FOLLOWX_MODE_DEFAULT,          "DEFAULT (follow_x_rules param)" },
        { FOLLOWX_MODE_SIM_DEFAULT,      "SIM_DEFAULT (opkey auto)" },
        { FOLLOWX_MODE_SIM_1,            "SIM_1 (opkey0)" },
        { FOLLOWX_MODE_SIM_2,            "SIM_2 (opkey1)" },
    };

    printf("\n=== File Lookup: %s ===\n", rel_path);
    for (size_t i = 0; i < sizeof(modes) / sizeof(modes[0]); i++) {
        char buf[MAX_PATH_LEN] = {0};
        char *found = GetOneCfgFileEx(rel_path, buf, MAX_PATH_LEN,
                                      modes[i].mode, NULL);
        printf("  mode=%-40s -> %s\n",
               modes[i].name,
               found ? found : "(not found)");
    }
}

static void print_usage(const char *prog)
{
    printf("Usage:\n");
    printf("  %s                    # show layer paths only\n", prog);
    printf("  %s <rel_path> ...     # also test file lookup\n", prog);
    printf("Example:\n");
    printf("  %s etc/telephony/OpkeyInfo.json\n", prog);
}

int main(int argc, char *argv[])
{
    if (argc >= 2 && (strcmp(argv[1], "-h") == 0 || strcmp(argv[1], "--help") == 0)) {
        print_usage(argv[0]);
        return 0;
    }

    dump_layers();

    for (int i = 1; i < argc; i++) {
        dump_file_lookup(argv[i]);
    }

    return 0;
}
```

**运行示例：**

```bash
# 只看分层路径
./dump_config_layers

# 同时诊断某文件在各 Follow-X 模式下的查找结果
./dump_config_layers etc/telephony/OpkeyInfo.json
```

**输出示例：**

```
=== Config Layer Paths ===
Raw policy value : /system:/chipset:/sys_prod:/chip_prod
Parsed paths (priority: low -> high):
  [0] /system
  [1] /chipset
  [2] /sys_prod
  [3] /chip_prod

=== File Lookup: etc/telephony/OpkeyInfo.json ===
  mode=NO_RULE_FOLLOWED                    -> /chip_prod/etc/telephony/OpkeyInfo.json
  mode=DEFAULT (follow_x_rules param)      -> /chip_prod/etc/carrier/46060/etc/telephony/OpkeyInfo.json
  mode=SIM_DEFAULT (opkey auto)            -> /chip_prod/etc/carrier/46060/etc/telephony/OpkeyInfo.json
  mode=SIM_1 (opkey0)                      -> /chip_prod/etc/carrier/46060/etc/telephony/OpkeyInfo.json
  mode=SIM_2 (opkey1)                      -> /chip_prod/etc/carrier/46061/etc/telephony/OpkeyInfo.json
```

### 3.3 方案三：将层级路径写入自有参数系统（进阶，按需）

若移植目标系统有自己的参数/属性系统（类似 Android 的 `getprop`），可在模块初始化时主动将实际生效路径写入参数系统，便于运维监控工具统一采集。

在适配层 `config_policy_param_adapter.h` 实现后，可在适配文件中添加：

```c
// 在参数系统初始化完成后调用一次
void ExportConfigPolicyLayersToParam(void)
{
    CfgDir *dirs = GetCfgDirList();
    if (dirs == NULL || dirs->realPolicyValue == NULL) return;

    // 写入自有参数系统，key 名称按本系统规范自定义
    YourSystemSetParameter("debug.cust.active_layers", dirs->realPolicyValue);

    FreeCfgDirList(dirs);
}
```

之后即可通过自有参数查询命令读取，无需专用二进制。

## 四、observability 核心点速查

| 观测目标 | 推荐方法 | 注意事项 |
|----------|----------|----------|
| 查看分层路径原始字符串 | `param get const.cust.config_dir_layer` | 参数未设置时返回空，实际仍有默认路径生效 |
| 查看解析后实际生效路径 | 运行 `dump_config_layers` 工具 | 读取 `realPolicyValue` 字段，不受盲区影响 |
| 诊断文件查找路径 | `dump_config_layers <rel_path>` | 覆盖所有 Follow-X 模式 |
| 查看 Follow-X 规则配置 | `param get const.cust.follow_x_rules` | 格式：`:relPath,mode[,extra][:]` |
| 查看 SIM 运营商 opkey | `param get telephony.sim.opkey0` / `opkey1` | 影响 SIM 模式路径拼接 |
| 确认模块加载 | 查看 HiLog：domain=`0xD001E00` | 仅有错误日志，无路径内容 |
