# config_policy 环境变量替代系统参数适配指导

## 适用场景

移植目标系统**没有**类似 OpenHarmony `param get` 或 Android `getprop` 的系统参数/属性系统，
使用标准 POSIX 环境变量作为配置分层路径的存储和传递机制。

---

## 一、实现原理

### 1.1 参数系统的访问路径

`config_policy_utils.c` 对参数系统的所有访问都通过内部函数 `CustGetSystemParam()` 完成，
而该函数依赖 `init_param.h` 提供的 `SystemGetParameter()`。
整个调用链如下：

```
GetCfgDirRealPolicyValue()  ──┐
GetFollowXRule()             ──┼──> CustGetSystemParam() ──> SystemGetParameter()
GetOpkeyPath()               ──┘                              （来自 init_param.h）
```

只要在 `config_policy_param_adapter.h` 中用 `getenv()` 实现 `SystemGetParameter()`，
业务代码**零改动**。

### 1.2 与 `移植指导.md` 的关系

本文档是 `移植指导.md` 中"适配方案 A（同名头文件包装）"的具体实现。
`config_policy_utils.c` 中只需改动 2 处 `#include`（移植指导中已说明），
本文档聚焦在适配头文件的完整实现。

### 1.3 `SystemGetParameter` 的两步调用模式

源码中固定使用如下两步模式（`config_policy_utils.c:107`）：

```c
unsigned int len = 0;

/* 第 1 步：传 NULL buffer，获取所需缓冲区长度 */
if (SystemGetParameter(name, NULL, &len) != 0 || len <= 0) {
    return NULL;
}

/* 第 2 步：分配内存，再次调用取值 */
value = (char *)calloc(len, sizeof(char));
if (value != NULL && SystemGetParameter(name, value, &len) == 0 && value[0]) {
    return value;
}
```

`getenv()` 本身不支持这种两步调用，但在适配层模拟很简单：
- 第 1 步：用 `strlen(getenv(...)) + 1` 计算长度写入 `*len`，返回 0
- 第 2 步：将值 `memcpy` 进调用方分配的缓冲区，返回 0

## 二、环境变量命名规则

将参数 key 中的 `.` 和 `-` 替换为 `_`，全部转大写：

| 系统参数 key | 对应环境变量名 | 说明 |
|---|---|---|
| `const.cust.config_dir_layer` | `CONST_CUST_CONFIG_DIR_LAYER` | 配置分层路径（核心） |
| `const.cust.follow_x_rules` | `CONST_CUST_FOLLOW_X_RULES` | Follow-X 规则（可选） |
| `telephony.sim.opkey0` | `TELEPHONY_SIM_OPKEY0` | SIM 卡1运营商 opkey（可选） |
| `telephony.sim.opkey1` | `TELEPHONY_SIM_OPKEY1` | SIM 卡2运营商 opkey（可选） |

转换逻辑完全自动，无需手写映射表，适配头文件内置转换函数。

## 三、适配头文件完整实现

新建文件 `port/config_policy_param_adapter.h`，内容如下：

```c
/*
 * config_policy_param_adapter.h
 *
 * 用标准 POSIX 环境变量替代 OpenHarmony init_param.h，
 * 适用于没有系统参数/属性系统的移植目标平台。
 *
 * 参数名转换规则：将 '.' 和 '-' 替换为 '_'，全部转大写
 *   e.g. "const.cust.config_dir_layer" -> "CONST_CUST_CONFIG_DIR_LAYER"
 */
#ifndef CONFIG_POLICY_PARAM_ADAPTER_H
#define CONFIG_POLICY_PARAM_ADAPTER_H

#include <ctype.h>
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

#ifdef __cplusplus
extern "C" {
#endif

/*
 * PARAM_CONST_VALUE_LEN_MAX
 *
 * 必须与 OH 原始定义兼容，源码用它限制动态缓冲区上限（config_policy_utils.c:298）。
 * OH 原始值为 96，但环境变量存储的路径字符串可能更长，
 * 此处扩大到 512 以适应多层自定义路径场景。
 * 如实际路径字符串更长，请相应调大此值。
 */
#ifndef PARAM_CONST_VALUE_LEN_MAX
#define PARAM_CONST_VALUE_LEN_MAX 512
#endif

/* ------------------------------------------------------------------ */
/* 内部工具：参数名 -> 环境变量名                                       */
/* ------------------------------------------------------------------ */

/**
 * 将参数 key 转换为环境变量名。
 * 规则：'.' '-' ':' 替换为 '_'，其余字符转大写。
 * buf 至少需要 strlen(name)+1 字节。
 */
static inline void _param_name_to_env(const char *name, char *buf, size_t buf_size)
{
    size_t i = 0;
    for (; i < buf_size - 1 && name[i] != '\0'; i++) {
        unsigned char c = (unsigned char)name[i];
        if (c == '.' || c == '-' || c == ':') {
            buf[i] = '_';
        } else {
            buf[i] = (char)toupper(c);
        }
    }
    buf[i] = '\0';
}

/* ------------------------------------------------------------------ */
/* SystemGetParameter                                                   */
/* ------------------------------------------------------------------ */

/**
 * 从环境变量读取参数，接口与 OpenHarmony init_param.h 完全一致。
 *
 * @param name   参数 key（如 "const.cust.config_dir_layer"）
 * @param value  输出缓冲区；传 NULL 时仅查询所需长度
 * @param len    输入/输出：
 *               value==NULL 时：写入所需缓冲区字节数（含 '\0'），返回 0
 *               value!=NULL 时：作为缓冲区大小输入，值复制成功返回 0
 * @return 0 成功，非零失败（参数不存在或缓冲区不足）
 */
static inline int SystemGetParameter(const char *name, char *value, unsigned int *len)
{
    if (name == NULL || len == NULL) {
        return -1;
    }

    /* 转换为环境变量名 */
    char env_name[256];
    _param_name_to_env(name, env_name, sizeof(env_name));

    const char *env_val = getenv(env_name);
    if (env_val == NULL) {
        return -1; /* 环境变量不存在，等价于参数未设置 */
    }

    unsigned int val_len = (unsigned int)(strlen(env_val) + 1); /* 含 '\0' */

    if (value == NULL) {
        /* 第 1 步调用：只返回所需长度 */
        *len = val_len;
        return 0;
    }

    /* 第 2 步调用：复制值 */
    if (*len < val_len) {
        return -1; /* 调用方提供的缓冲区不足 */
    }
    memcpy(value, env_val, val_len);
    return 0;
}

/* ------------------------------------------------------------------ */
/* SystemSetParameter（可选，主要用于单元测试）                         */
/* ------------------------------------------------------------------ */

/**
 * 将参数写入环境变量。
 * 注意：setenv() 修改当前进程的环境，不会持久化到系统。
 * 主要用于测试时动态覆盖配置，生产环境通过 shell export 或 init 脚本设置。
 *
 * @return 0 成功，非零失败
 */
static inline int SystemSetParameter(const char *name, const char *value)
{
    if (name == NULL || value == NULL) {
        return -1;
    }
    char env_name[256];
    _param_name_to_env(name, env_name, sizeof(env_name));

#if defined(_WIN32)
    return _putenv_s(env_name, value) == 0 ? 0 : -1;
#else
    return setenv(env_name, value, 1 /* overwrite */) == 0 ? 0 : -1;
#endif
}

#ifdef __cplusplus
}
#endif

#endif /* CONFIG_POLICY_PARAM_ADAPTER_H */
```

## 五、设置环境变量的方式

### 5.1 shell 临时设置（开发调试）

仅对当前 shell 会话生效：

```bash
export CONST_CUST_CONFIG_DIR_LAYER="/system:/vendor:/oem"
export CONST_CUST_FOLLOW_X_RULES=""
export TELEPHONY_SIM_OPKEY0="46060"
export TELEPHONY_SIM_OPKEY1="46061"
```

仅对单次命令生效：

```bash
CONST_CUST_CONFIG_DIR_LAYER="/system:/vendor:/oem" ./your_program
```

### 5.2 全局持久化（/etc/environment）

适用于大多数 Linux 发行版，重启后生效：

```bash
# /etc/environment
# 注意：此文件不是 shell 脚本，不支持 export 关键字和变量展开
CONST_CUST_CONFIG_DIR_LAYER=/system:/vendor:/oem
CONST_CUST_FOLLOW_X_RULES=
TELEPHONY_SIM_OPKEY0=46060
TELEPHONY_SIM_OPKEY1=46061
```

### 5.3 systemd 服务（推荐用于守护进程）

```ini
# /etc/systemd/system/your-service.service
[Service]
Environment="CONST_CUST_CONFIG_DIR_LAYER=/system:/vendor:/oem"
Environment="CONST_CUST_FOLLOW_X_RULES=:etc/telephony/OpkeyInfo.json,10"
Environment="TELEPHONY_SIM_OPKEY0=46060"
Environment="TELEPHONY_SIM_OPKEY1=46061"
ExecStart=/usr/bin/your_program
```

也可以将环境变量单独写入文件，通过 `EnvironmentFile` 引用：

```ini
[Service]
EnvironmentFile=/etc/your-service/config-policy.env
ExecStart=/usr/bin/your_program
```

```bash
# /etc/your-service/config-policy.env
CONST_CUST_CONFIG_DIR_LAYER=/system:/vendor:/oem
TELEPHONY_SIM_OPKEY0=46060
```

### 5.4 传统 init 脚本（SysV init / busybox init）

```sh
#!/bin/sh
# /etc/init.d/your-service

export CONST_CUST_CONFIG_DIR_LAYER="/system:/vendor:/oem"
export TELEPHONY_SIM_OPKEY0="46060"
export TELEPHONY_SIM_OPKEY1="46061"

exec /usr/bin/your_program
```

### 5.5 目标系统自有 init 进程（鸿蒙内核场景）

若目标系统有自己的 init 进程但没有参数系统，可在进程启动前通过 `putenv()` 或 `setenv()` 完成初始化，或在 init 脚本中 `export` 后再 `exec` 子进程。环境变量会通过 `fork()`/`exec()` 自动继承到子进程，无需额外传递机制。

## 六、注意事项

### 6.1 路径值不需要 shell 引号

环境变量值本身不含 shell 引号，冒号分隔符直接写入：

```bash
# 正确
export CONST_CUST_CONFIG_DIR_LAYER=/system:/vendor:/oem

# 错误（引号会被当作路径字符的一部分）
export CONST_CUST_CONFIG_DIR_LAYER='"/system:/vendor:/oem"'
```

在 `/etc/environment` 文件中，等号右侧的引号是可选的且通常不需要，具体取决于系统对该文件的解析方式。

### 6.2 `PARAM_CONST_VALUE_LEN_MAX` 须覆盖实际路径长度

`ExpandStr()` 函数（`config_policy_utils.c:298`）用此值限制动态缓冲区上限：

```c
size_t allocSize = Min(Max(leastSize * 2, MIN_APPEND_LEN), PARAM_CONST_VALUE_LEN_MAX);
```

若分层路径字符串总长度超过此值，`ExpandStr()` 会截断失败。
请根据实际路径长度调整适配头文件中的定义：

```c
// 路径层数多、路径名长时适当调大
#define PARAM_CONST_VALUE_LEN_MAX 1024
```

### 6.3 `getenv()` 的线程安全边界

| 场景 | 是否安全 |
|------|----------|
| 多线程并发**读取**环境变量（无写操作） | 安全 |
| 读取与 `setenv()`/`putenv()` 并发 | **未定义行为（UB）** |

典型使用场景（程序启动前配置好环境变量，运行期不再修改）是安全的。
若需要运行期动态修改分层路径，应改用带锁的自定义参数存储，不要依赖环境变量。

### 6.4 环境变量的进程继承

环境变量在 `fork()` 时自动复制给子进程。
若程序会启动子进程且不希望配置信息泄漏，在 `exec()` 前手动清除：

```c
unsetenv("CONST_CUST_CONFIG_DIR_LAYER");
unsetenv("CONST_CUST_FOLLOW_X_RULES");
```

### 6.5 环境变量未设置时的行为

与 OH 参数系统行为完全一致：`getenv()` 返回 `NULL`，
`SystemGetParameter()` 返回非零，`CustGetSystemParam()` 返回 `NULL`，
模块最终静默回落到编译期 `DEFAULT_LAYER`。无需额外处理。

## 七、CMakeLists.txt 集成

```cmake
cmake_minimum_required(VERSION 3.14)
project(config_policy_util C)

# 适配头文件目录（包含本文档的 config_policy_param_adapter.h）
set(PORT_INCLUDE_DIR ${CMAKE_SOURCE_DIR}/port)

add_library(config_policy_util SHARED
    frameworks/config_policy/src/config_policy_utils.c
)

target_include_directories(config_policy_util PUBLIC
    interfaces/inner_api/include   # config_policy_utils.h、config_policy_impl.h
    ${PORT_INCLUDE_DIR}            # config_policy_param_adapter.h、compat_securec.h
)

target_compile_options(config_policy_util PRIVATE
    -std=c11
    -D_POSIX_C_SOURCE=200809L      # 启用 strtok_r、setenv 等 POSIX 扩展
    -Wall
    -Wextra
)

# 可选：构建诊断工具（参见可观测性指导.md）
add_executable(dump_config_layers tools/dump_config_layers.c)
target_link_libraries(dump_config_layers config_policy_util)
target_include_directories(dump_config_layers PRIVATE
    interfaces/inner_api/include
)
```
