## 四、发现的代码缺陷（不影响兼容性，但需修正）

### 缺陷1：`buildDirList` 是死代码，应删除

**位置**：`config_policy.cpp` 匿名命名空间

**问题代码**：

```cpp
ConfigDirList buildDirList(const CfgDirPtr& dirs)
{
    std::vector<std::string> paths;
    paths.reserve(MAX_CFG_POLICY_DIRS_CNT);
    for (int i = 0; i < MAX_CFG_POLICY_DIRS_CNT; ++i) {
        if (dirs->paths[i] != nullptr) {
            paths.emplace_back(dirs->paths[i]);
        }
    }
    std::string rawValue =
        (dirs->realPolicyValue != nullptr) ? dirs->realPolicyValue : "";

    (void)rawValue;  // ← 计算结果被丢弃
    (void)paths;     // ← 计算结果被丢弃
    return {};       // ← 只返回默认空对象，所有计算作废
}
```

**根本原因**：`ConfigDirList` 的私有构造函数声明了 `friend class ConfigPolicy`，
只有 `ConfigPolicy` 的成员函数才能调用它。
匿名命名空间中的自由函数不属于 `ConfigPolicy`，**没有 friend 访问权**，
无法调用私有构造函数构造出有效的 `ConfigDirList`。
作者意识到这一点后，用 `(void)` 丢弃所有计算结果并 `return {}` 放弃，
导致整个函数成为无效代码。

**实际逻辑**：正确的拷贝逻辑已在 `ConfigPolicy::getDirList()` 内部直接实现（是正确的）。`buildDirList` 从未被调用，应整体删除。

**修正方案**：

```diff
 namespace {

 using CfgDirPtr   = std::unique_ptr<CfgDir,   decltype(&FreeCfgDirList)>;
 using CfgFilesPtr = std::unique_ptr<CfgFiles,  decltype(&FreeCfgFiles)>;

-/// 将 CfgDir* 的内容急迫拷贝为 ConfigDirList（值类型）
-/// 调用前 dirs 必须非空
-ConfigDirList buildDirList(const CfgDirPtr& dirs)
-{
-    std::vector<std::string> paths;
-    paths.reserve(MAX_CFG_POLICY_DIRS_CNT);
-    for (int i = 0; i < MAX_CFG_POLICY_DIRS_CNT; ++i) {
-        if (dirs->paths[i] != nullptr) {
-            paths.emplace_back(dirs->paths[i]);
-        }
-    }
-    std::string rawValue =
-        (dirs->realPolicyValue != nullptr) ? dirs->realPolicyValue : "";
-    (void)rawValue;
-    (void)paths;
-    return {};
-}
-
 } // namespace (anonymous)
```

删除后 `getDirList()` 的实现无需任何改动，其内联逻辑本身是正确的。

### 缺陷2：`#include <cstring>` 冗余，应删除

**位置**：`config_policy.cpp` 头部

**问题**：`#include <cstring>` 隐含 `std::strlen` 的使用意图，
但最终实现中 `std::strlen` 实际上**从未被调用**。
`std::string` 的构造函数接受 `const char*` 时会自动计算长度，无需手动调用 `std::strlen`。

**修正方案**：

```diff
 #include <memory>   // std::unique_ptr
-#include <cstring>  // std::strlen
```
