# dump_config_layers 与 verify_port 动态链接改造指导

## 一、背景与结论

### 1.1 当前构建方式

| 工具 | 当前链接方式 | 编译时依赖 |
|------|-------------|-----------|
| `dump_config_layers` | 隐式动态链接（CMake `target_link_libraries`） | 编译时必须有 `libconfig_policy_util.so` |
| `verify_port` | 隐式动态链接（`gcc -lconfigpolicy_util`） | 编译时必须有 `libconfig_policy_util.so` |

**隐式动态链接**：链接器在编译时把符号绑定信息写入可执行文件，
运行时由动态链接器（`ld.so`）在 `LD_LIBRARY_PATH` 等标准路径中自动搜索并加载。

### 1.2 两种动态链接方式对比

| 对比项 | 隐式动态链接（当前） | `dlopen` 显式动态加载（改造后） |
|--------|---------------------|-------------------------------|
| 编译时依赖 | 必须能找到 `.so` | **不需要**，仅需头文件 |
| 运行时 `.so` 路径 | 依赖 `LD_LIBRARY_PATH` / `rpath` | **可在命令行参数中指定** |
| `.so` 缺失时行为 | 进程启动即崩溃（`error loading shared libraries`） | **主动报错并提示路径** |
| 指定测试不同版本 | 需要改环境变量 | `./tool /path/to/other_version.so` |
| 适合场景 | 应用程序（业务代码） | **诊断 / 验证工具** |

### 1.3 为什么改造更合适

两个工具的角色决定了 `dlopen` 更适合：

- **`dump_config_layers`** 是运维诊断工具，常在部署环境中独立运行，
  无法保证 `LD_LIBRARY_PATH` 已正确设置；
  使用 `dlopen` 可以直接传入 `.so` 的绝对路径，
  即使库不在标准路径也能正常工作。

- **`verify_port`** 的目的是验证移植结果，
  应该测试**实际部署的 `.so`**，而不是编译时恰好链接到的那一份；
  使用 `dlopen` 能明确指定被验证的目标文件，
  测试意图更清晰。

## 二、改造后的 `tools/dump_config_layers.c`

```c
/*
 * tools/dump_config_layers.c  —  配置分层路径诊断工具（dlopen 版）
 *
 * 编译：
 *   gcc dump_config_layers.c -I interfaces/inner_api/include -ldl -o dump_config_layers
 *
 * 用法：
 *   ./dump_config_layers [/path/to/libconfig_policy_util.so] [rel_path ...]
 *
 * 示例：
 *   ./dump_config_layers                                      # 搜索默认路径
 *   ./dump_config_layers ./libconfig_policy_util.so           # 指定 .so 路径
 *   ./dump_config_layers ./libconfig_policy_util.so etc/cfg.xml  # 同时诊断文件查找
 */

#include <stdio.h>
#include <string.h>
#include <dlfcn.h>
#include "config_policy_utils.h"   /* 仅用于结构体类型定义和常量，不依赖实现 */

/* ── 函数指针类型 ─────────────────────────────────────────────────── */
typedef CfgDir  *(*fn_GetCfgDirList)(void);
typedef void     (*fn_FreeCfgDirList)(CfgDir *);
typedef char    *(*fn_GetOneCfgFileEx)(const char *, char *, size_t, int,
                                        const char *);

/* ── 运行时句柄与函数指针 ──────────────────────────────────────────── */
static void               *g_handle          = NULL;
static fn_GetCfgDirList    g_GetCfgDirList   = NULL;
static fn_FreeCfgDirList   g_FreeCfgDirList  = NULL;
static fn_GetOneCfgFileEx  g_GetOneCfgFileEx = NULL;

/* ── 加载动态库 ────────────────────────────────────────────────────── */
static int load_library(const char *so_path)
{
    /* 清除上一次的 dlerror */
    dlerror();

    g_handle = dlopen(so_path, RTLD_LAZY);
    if (!g_handle) {
        fprintf(stderr, "[ERROR] dlopen(\"%s\") 失败: %s\n",
                so_path, dlerror());
        return -1;
    }

    g_GetCfgDirList   = (fn_GetCfgDirList)  dlsym(g_handle, "GetCfgDirList");
    g_FreeCfgDirList  = (fn_FreeCfgDirList) dlsym(g_handle, "FreeCfgDirList");
    g_GetOneCfgFileEx = (fn_GetOneCfgFileEx)dlsym(g_handle, "GetOneCfgFileEx");

    const char *err = dlerror();
    if (err || !g_GetCfgDirList || !g_FreeCfgDirList || !g_GetOneCfgFileEx) {
        fprintf(stderr, "[ERROR] dlsym 失败: %s\n", err ? err : "符号不存在");
        dlclose(g_handle);
        g_handle = NULL;
        return -1;
    }

    printf("[INFO]  已加载: %s\n\n", so_path);
    return 0;
}

/* ── 打印层路径信息 ────────────────────────────────────────────────── */
static void dump_layers(void)
{
    CfgDir *dirs = g_GetCfgDirList();
    if (!dirs) {
        fprintf(stderr, "[ERROR] GetCfgDirList() 返回 NULL\n");
        return;
    }
    printf("=== Config Layer Paths ===\n");
    printf("Raw policy value : %s\n",
           dirs->realPolicyValue ? dirs->realPolicyValue
                                 : "(empty — 使用编译期 DEFAULT_LAYER)");
    printf("Parsed paths (priority: low -> high):\n");
    for (size_t i = 0; i < MAX_CFG_POLICY_DIRS_CNT; i++) {
        if (dirs->paths[i] && dirs->paths[i][0] != '\0') {
            printf("  [%zu] %s\n", i, dirs->paths[i]);
        }
    }
    g_FreeCfgDirList(dirs);
}

/* ── 诊断指定相对路径在各 Follow-X 模式下的查找结果 ─────────────── */
static void dump_file_lookup(const char *rel_path)
{
    static const struct {
        int         mode;
        const char *name;
    } modes[] = {
        { FOLLOWX_MODE_NO_RULE_FOLLOWED, "NO_RULE_FOLLOWED" },
        { FOLLOWX_MODE_DEFAULT,          "DEFAULT          " },
        { FOLLOWX_MODE_SIM_DEFAULT,      "SIM_DEFAULT      " },
        { FOLLOWX_MODE_SIM_1,            "SIM_1            " },
        { FOLLOWX_MODE_SIM_2,            "SIM_2            " },
    };
    const size_t n = sizeof(modes) / sizeof(modes[0]);

    printf("\n=== File Lookup: %s ===\n", rel_path);
    for (size_t i = 0; i < n; i++) {
        char buf[MAX_PATH_LEN] = {0};
        char *found = g_GetOneCfgFileEx(rel_path, buf, MAX_PATH_LEN,
                                         modes[i].mode, "");
        printf("  mode=%-20s -> %s\n",
               modes[i].name,
               found ? found : "(not found)");
    }
}

/* ── 打印用法 ──────────────────────────────────────────────────────── */
static void print_usage(const char *prog)
{
    printf("用法:\n");
    printf("  %s [so_path] [rel_path ...]\n\n", prog);
    printf("  so_path   libconfig_policy_util.so 的路径\n");
    printf("            省略时依次尝试:\n");
    printf("              ./libconfig_policy_util.so\n");
    printf("              libconfig_policy_util.so (LD_LIBRARY_PATH)\n\n");
    printf("  rel_path  可选，对指定相对路径诊断所有 Follow-X 模式的查找结果\n\n");
    printf("示例:\n");
    printf("  %s\n", prog);
    printf("  %s /usr/lib/libconfig_policy_util.so\n", prog);
    printf("  %s ./libconfig_policy_util.so etc/telephony/OpkeyInfo.json\n", prog);
}

/* ── 入口 ──────────────────────────────────────────────────────────── */
int main(int argc, char *argv[])
{
    if (argc >= 2 &&
        (strcmp(argv[1], "-h") == 0 || strcmp(argv[1], "--help") == 0)) {
        print_usage(argv[0]);
        return 0;
    }

    /* 判断第一个参数是 .so 路径还是 rel_path */
    int    path_args_start = 1; /* rel_path 参数从哪个 argv 索引开始 */
    const char *so_path    = NULL;

    if (argc >= 2) {
        const char *a = argv[1];
        size_t      alen = strlen(a);
        /* 若参数以 .so 结尾或包含 '/'，视为 so_path */
        if ((alen >= 3 && strcmp(a + alen - 3, ".so") == 0) ||
            strchr(a, '/') != NULL) {
            so_path         = a;
            path_args_start = 2;
        }
    }

    /* 尝试加载库 */
    if (so_path) {
        if (load_library(so_path) != 0) return 1;
    } else {
        /* 依次尝试本地路径和系统路径 */
        if (load_library("./libconfig_policy_util.so") != 0 &&
            load_library("libconfig_policy_util.so")   != 0) {
            fprintf(stderr,
                    "[HINT]  请通过参数指定 .so 路径:\n"
                    "        %s /path/to/libconfig_policy_util.so\n",
                    argv[0]);
            return 1;
        }
    }

    /* 打印层路径 */
    dump_layers();

    /* 对每个指定的相对路径诊断文件查找结果 */
    for (int i = path_args_start; i < argc; i++) {
        dump_file_lookup(argv[i]);
    }

    dlclose(g_handle);
    return 0;
}
```

## 三、改造后的 `tools/verify_port.c`

```c
/*
 * tools/verify_port.c  —  移植验证程序（dlopen 版）
 *
 * 编译：
 *   gcc verify_port.c -I interfaces/inner_api/include -ldl -o verify_port
 *
 * 用法：
 *   ./verify_port [/path/to/libconfig_policy_util.so]
 *
 * 验证项目：
 *   1. 动态加载 .so 成功
 *   2. 所有公共 API 符号可解析
 *   3. GetCfgDirList() 返回非空结果
 *   4. GetOneCfgFile() 对不存在路径返回 NULL（不崩溃）
 *   5. NULL 参数不导致崩溃
 *   6. Free 函数正常工作（Valgrind 无泄漏）
 */

#include <stdio.h>
#include <string.h>
#include <dlfcn.h>
#include "config_policy_utils.h"

/* ── 函数指针类型 ─────────────────────────────────────────────────── */
typedef CfgDir   *(*fn_GetCfgDirList)(void);
typedef void      (*fn_FreeCfgDirList)(CfgDir *);
typedef CfgFiles *(*fn_GetCfgFilesEx)(const char *, int, const char *);
typedef void      (*fn_FreeCfgFiles)(CfgFiles *);
typedef char     *(*fn_GetOneCfgFileEx)(const char *, char *, size_t, int,
                                         const char *);

/* ── 结果统计 ─────────────────────────────────────────────────────── */
static int g_pass = 0;
static int g_fail = 0;

#define CHECK(label, expr)                                          \
    do {                                                            \
        if (expr) {                                                 \
            printf("  [PASS] %s\n", (label));                      \
            g_pass++;                                               \
        } else {                                                    \
            printf("  [FAIL] %s\n", (label));                      \
            g_fail++;                                               \
        }                                                           \
    } while (0)

/* ── 入口 ──────────────────────────────────────────────────────────── */
int main(int argc, char *argv[])
{
    const char *so_path = (argc >= 2) ? argv[1] : "libconfig_policy_util.so";

    printf("===== verify_port: 移植验证 =====\n");
    printf("目标库: %s\n\n", so_path);

    /* ── 验证1：动态加载 ── */
    printf("[ 验证1 ] 动态加载\n");
    dlerror();
    void *handle = dlopen(so_path, RTLD_LAZY);
    CHECK("dlopen() 成功", handle != NULL);
    if (!handle) {
        fprintf(stderr, "  原因: %s\n", dlerror());
        printf("\n结果: 0 passed, 1 failed — 无法继续\n");
        return 1;
    }

    /* ── 验证2：符号解析 ── */
    printf("\n[ 验证2 ] 公共 API 符号解析\n");
    fn_GetCfgDirList   GetCfgDirList   =
        (fn_GetCfgDirList)  dlsym(handle, "GetCfgDirList");
    fn_FreeCfgDirList  FreeCfgDirList  =
        (fn_FreeCfgDirList) dlsym(handle, "FreeCfgDirList");
    fn_GetCfgFilesEx   GetCfgFilesEx   =
        (fn_GetCfgFilesEx)  dlsym(handle, "GetCfgFilesEx");
    fn_FreeCfgFiles    FreeCfgFiles    =
        (fn_FreeCfgFiles)   dlsym(handle, "FreeCfgFiles");
    fn_GetOneCfgFileEx GetOneCfgFileEx =
        (fn_GetOneCfgFileEx)dlsym(handle, "GetOneCfgFileEx");

    CHECK("GetCfgDirList   符号存在", GetCfgDirList   != NULL);
    CHECK("FreeCfgDirList  符号存在", FreeCfgDirList  != NULL);
    CHECK("GetCfgFilesEx   符号存在", GetCfgFilesEx   != NULL);
    CHECK("FreeCfgFiles    符号存在", FreeCfgFiles    != NULL);
    CHECK("GetOneCfgFileEx 符号存在", GetOneCfgFileEx != NULL);

    if (!GetCfgDirList || !FreeCfgDirList ||
        !GetCfgFilesEx || !FreeCfgFiles   || !GetOneCfgFileEx) {
        printf("\n结果: %d passed, %d failed — 符号缺失，停止验证\n",
               g_pass, g_fail);
        dlclose(handle);
        return 1;
    }

    /* ── 验证3：GetCfgDirList 基本功能 ── */
    printf("\n[ 验证3 ] GetCfgDirList() 基本功能\n");
    CfgDir *dirs = GetCfgDirList();
    CHECK("GetCfgDirList() 返回非 NULL", dirs != NULL);
    if (dirs) {
        int has_path = 0;
        for (size_t i = 0; i < MAX_CFG_POLICY_DIRS_CNT; i++) {
            if (dirs->paths[i] && dirs->paths[i][0] != '\0') {
                has_path = 1;
                break;
            }
        }
        CHECK("paths[] 至少有一条非空路径", has_path);
        CHECK("realPolicyValue 非 NULL", dirs->realPolicyValue != NULL);

        printf("  实际层路径: %s\n",
               dirs->realPolicyValue ? dirs->realPolicyValue : "(null)");
        FreeCfgDirList(dirs);
        CHECK("FreeCfgDirList() 不崩溃", 1);
    }

    /* ── 验证4：GetOneCfgFile 对不存在文件返回 NULL ── */
    printf("\n[ 验证4 ] GetOneCfgFileEx() 边界行为\n");
    char buf[MAX_PATH_LEN] = {0};
    char *found = GetOneCfgFileEx(
        "__nonexistent_verify_port_file_xyz__",
        buf, MAX_PATH_LEN,
        FOLLOWX_MODE_NO_RULE_FOLLOWED, "");
    CHECK("查询不存在文件返回 NULL（不崩溃）", found == NULL);

    /* ── 验证5：GetCfgFilesEx 对不存在文件返回空/NULL ── */
    printf("\n[ 验证5 ] GetCfgFilesEx() 边界行为\n");
    CfgFiles *files = GetCfgFilesEx(
        "__nonexistent_verify_port_file_xyz__",
        FOLLOWX_MODE_NO_RULE_FOLLOWED, "");
    /* 返回 NULL 或 paths 全为空均可接受 */
    int files_ok = 1;
    if (files) {
        for (size_t i = 0; i < MAX_CFG_POLICY_DIRS_CNT; i++) {
            if (files->paths[i] && strstr(files->paths[i], "nonexistent")) {
                files_ok = 0;
                break;
            }
        }
        FreeCfgFiles(files);
    }
    CHECK("查询不存在文件不返回错误路径", files_ok);

    /* ── 验证6：FreeCfgDirList(NULL) 不崩溃 ── */
    printf("\n[ 验证6 ] NULL 参数安全性\n");
    FreeCfgDirList(NULL);
    CHECK("FreeCfgDirList(NULL) 不崩溃", 1);
    FreeCfgFiles(NULL);
    CHECK("FreeCfgFiles(NULL) 不崩溃", 1);

    /* ── 收尾 ── */
    dlclose(handle);
    printf("\n===== 验证结果: %d passed, %d failed =====\n",
           g_pass, g_fail);
    return g_fail > 0 ? 1 : 0;
}
```

## 四、CMakeLists.txt 修改

```cmake
# ── 诊断工具 dump_config_layers ──────────────────────────────────────
add_executable(dump_config_layers tools/dump_config_layers.c)

target_include_directories(dump_config_layers PRIVATE
    interfaces/inner_api/include   # config_policy_utils.h（仅类型定义）
)

target_link_libraries(dump_config_layers
    dl                             # dlopen / dlsym / dlclose
    # 注意：不再链接 config_policy_util
    # 运行时通过 dlopen 加载，编译时无依赖
)

target_compile_options(dump_config_layers PRIVATE -Wall -Wextra)


# ── 移植验证工具 verify_port ─────────────────────────────────────────
add_executable(verify_port tools/verify_port.c)

target_include_directories(verify_port PRIVATE
    interfaces/inner_api/include   # config_policy_utils.h（仅类型定义）
)

target_link_libraries(verify_port
    dl                             # dlopen / dlsym / dlclose
    # 注意：不再链接 config_policy_util
)

target_compile_options(verify_port PRIVATE -Wall -Wextra)
```

**与改造前 CMakeLists.txt 的差异：**

```
改造前：
  target_link_libraries(dump_config_layers config_policy_util)
  target_link_libraries(verify_port        config_policy_util)

改造后：
  target_link_libraries(dump_config_layers dl)
  target_link_libraries(verify_port        dl)
```

## 六、注意事项

### 6.1 `config_policy_utils.h` 仍需包含

头文件仅提供**结构体类型**（`CfgDir`、`CfgFiles`）和**常量**
（`MAX_CFG_POLICY_DIRS_CNT`、`MAX_PATH_LEN`、`FOLLOWX_MODE_*`），
本身不依赖任何 OpenHarmony 专有头文件，可以直接包含。

工具需要这些类型才能解引用 `CfgDir*` 和 `CfgFiles*`。

### 6.2 函数指针强制转换警告

`dlsym` 返回 `void*`，转换为函数指针在 C99 中是未定义行为
（但在 POSIX 系统上被明确允许，且所有主流编译器均支持）。
若编译出现警告，可用如下方式消除：

```c
/* GCC / Clang 推荐写法，利用联合体绕过警告 */
union { void *ptr; fn_GetCfgDirList fn; } u;
u.ptr = dlsym(handle, "GetCfgDirList");
g_GetCfgDirList = u.fn;
```

### 6.3 macOS 的 `.dylib`

macOS 上动态库扩展名为 `.dylib`，`dlopen` 用法相同，
`CMakeLists.txt` 中 `dl` 库不需要（macOS 默认链接 libSystem）：

```cmake
if(NOT APPLE)
    target_link_libraries(dump_config_layers dl)
endif()
```

### 6.4 `FreeCfgDirList(NULL)` 的防御性检查

验证项目6 假设 `FreeCfgDirList(NULL)` 不崩溃。
原始代码 `config_policy_utils.c` 中 `FreeCfgDirList` 有 NULL 检查，
移植后若有改动请确认此行为保持一致。

## 七、改造前后对比总结

| 项目 | 改造前（隐式链接） | 改造后（dlopen） |
|------|-------------------|-----------------|
| 编译时 `.so` 依赖 | 必须存在 | 不需要 |
| 运行时 `.so` 搜索 | `LD_LIBRARY_PATH` / `rpath` | 命令行参数指定 |
| `.so` 不存在时 | 启动即崩溃 | 主动报错并提示 |
| 测试特定 `.so` 版本 | 需改环境变量 | 直接传路径参数 |
| 适合角色 | 业务应用 | **诊断/验证工具** ✓ |
| 额外链接库 | 无 | `-ldl` |
| `.c` 文件改动 | — | 改用函数指针调用 |
